RDF Utility Functions

rewire_graph(g: Graph, mset: MappingSetDataFrame, subject_to_object: bool = True, precedence: List[str] | None = None) → int:

This function rewrites an RDF Graph by replacing nodes using equivalence mappings provided in a MappingSetDataFrame. The function can either map from subject to object (default behavior) or from object to subject. In case of ambiguity, precedence can be defined with a list of prefixes.

Parameters:

  • g (Graph) – The RDF graph to be rewired.

  • mset (MappingSetDataFrame) – The MappingSetDataFrame containing the equivalence mappings.

  • subject_to_object (bool, optional) – A boolean that determines the direction of the rewire. If true, the rewire is from subject to object. If false, the rewire is from object to subject. Default is True.

  • precedence (list, optional) – A list of prefixes to resolve any ambiguity in the mapping. If not provided, the function raises a ValueError when it encounters ambiguity. Default is None.

Returns:

The number of triples in the graph that were changed.

Return type:

int

Raises:

  • TypeError – If a mapping does not contain a valid subject or object.

  • ValueError – If there is ambiguity in the mapping and no precedence is defined.

rewire_node(n: Any)

This function is a helper function used to rewire individual nodes in the graph.

Parameters:

n (Any) – The node in the graph to be rewired.

Returns:

The rewired node. If the node is not in the rewire map, it returns the original node.

Return type:

Any
